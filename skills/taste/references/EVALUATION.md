# Evaluation

Use this rubric to score created work. The rubric tests for domain-grounded judgment, not generic polish.

---

## Dimensions

### 1. Domain Grounding (most important)

Does the output reflect actual understanding of this specific domain and context, or could it have been produced by an agent with no domain knowledge running a quality checklist?

- **5** — The choices reflect practitioner-level understanding. Someone in the field would recognize this as informed work.
- **3** — Generally appropriate, but some choices feel generic or borrowed from adjacent domains.
- **1** — Pattern-matched. Surface features are correct, but the judgment calls are obviously ungrounded.

### 2. Signal-to-Noise

Does every element serve the work, or is there material included because it was available?

- **5** — Nothing could be removed without loss. The hierarchy is clear and intentional.
- **3** — Core is solid, but there's residual padding or equal-weight treatment of unequal ideas.
- **1** — No hierarchy. Material included by availability, not by judgment.

### 3. Spine

Is there a controlling idea, and does the structure reinforce it?

- **5** — One clear thing the work is doing. Everything supports or gets out of its way.
- **3** — Identifiable center, but secondary ideas compete for attention.
- **1** — No clear spine. The work is a collection of elements, not a coherent creation.

### 4. Calibrated Finish

Is the level of polish appropriate to the context — neither undercooked nor overworked?

- **5** — The finish matches the stakes and the audience. Effort was spent where it matters.
- **3** — Slightly over- or under-finished for the context.
- **1** — Dramatically miscalibrated — either a rough draft when polish was needed, or obsessively refined when a working version would do.

### 5. Specificity

Is the work anchored in concrete, domain-real details — or is it operating in generalities?

- **5** — Concrete throughout. Real names, real patterns, real constraints from this actual context.
- **3** — Mix of specific and generic. Some grounded detail, some filler.
- **1** — Entirely generic. Could be about any project, any company, any problem in the category.

### 6. Register

Does the output sound like it belongs in this domain — not performing the domain, but native to it?

- **5** — A practitioner would read this without friction. The vocabulary, pacing, and emphasis feel natural.
- **3** — Mostly right, but occasional notes that feel borrowed or performed.
- **1** — Wrong register. Tutorial tone in a professional context, or formal tone in a casual one.

---

## Fast Check

Before delivering, ask these five questions. If any answer is "no," revise.

1. Did you ground in the domain before creating, or did you start from a generic template?
2. Is there at least one thing you excluded that a less informed agent would have included?
3. Can you state the spine in one sentence?
4. Would a practitioner in this domain consider this work informed?
5. Is the level of finish appropriate to the context — not maximal, but right?

---

## Scoring

If using the rubric formally:

- **27-30**: Strong taste. The work reflects genuine domain judgment.
- **21-26**: Good. Some generic residue, but the core judgment is sound.
- **15-20**: Adequate. The work is competent but doesn't demonstrate taste beyond slop removal.
- **Below 15**: The taste filter wasn't meaningfully applied.

The single most diagnostic dimension is **Domain Grounding**. If that score is low, the others are unreliable — the work might score well on Signal-to-Noise and Spine by coincidence rather than by judgment.
